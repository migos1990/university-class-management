**routes/classes.js:**

Add a `GET /` handler BEFORE the existing `GET /:id` handler. This should redirect to `/dashboard` since the dashboard already shows the class list. This is the simplest fix that resolves the 404.

```javascript
/**
 * GET /classes
 * Redirect to dashboard (which shows class list by role)
 */
router.get('/', requireAuth, (req, res) => {
  res.redirect('/dashboard');
});
```

Place this route BEFORE the `GET /:id` route (before line 12). The redirect is preferred over creating a new view because:
1. The student dashboard already shows enrolled classes
2. The professor dashboard already shows all classes
3. Creating a duplicate view adds maintenance burden
