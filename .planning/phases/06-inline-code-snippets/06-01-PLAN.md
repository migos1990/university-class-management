**CRITICAL XSS NOTE:** Use `<%= %>` (escaped) for the code_snippet output, NOT `<%- %>`. EJS auto-escapes `<`, `>`, `&` to HTML entities. Prism.js reads `textContent` from the DOM, which converts entities back to characters for correct tokenization. This is the safe and correct approach.

**CRITICAL FORMATTING NOTE:** The `<pre>` opening tag and `<code>` opening tag must be on the same line with NO whitespace before the `<%= finding.code_snippet %>`. Any whitespace between `<code ...>` and the content becomes visible in the rendered output. Similarly, the closing `</code></pre>` must immediately follow the content with no newline.
